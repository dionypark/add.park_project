# Amazon EC2 개요

Amazon EC2(Elastic Compute Cloud)는 가상 서버(인스턴스)를 시간 단위 또는 초 단위로 빌려 쓰는 IaaS 서비스다. 운영체제, 런타임, 애플리케이션까지 사용자가 직접 관리해야 하며, 그만큼 구성 자유도가 높다.

## 언제 적합한가

- 상시 실행되는 백엔드 서버, 장시간 실행되는 배치 작업, 상태를 오래 유지해야 하는 워크로드에 적합하다.
- 트래픽이 예측 가능하고 어느 정도 꾸준히 발생하는 경우, 서버가 계속 켜져 있는 비용이 상대적으로 저렴해질 수 있다.
- 커스텀 커널 모듈, 특정 런타임 버전, GPU 등 세밀한 인프라 제어가 필요할 때 적합하다.

## 요금 구조

- On-Demand: 사용한 시간만큼 지불. 트래픽이 적어도 인스턴스를 켜두면 계속 과금된다.
- Reserved Instance / Savings Plans: 1년/3년 약정으로 On-Demand 대비 최대 70%까지 할인. 사용량이 꾸준한 워크로드에 유리.
- Spot Instance: 유휴 용량을 최대 90% 할인된 가격에 사용하지만 언제든 회수(interrupt)될 수 있어 상태 없는(stateless) 배치/실험 작업에 적합.
- 인스턴스가 꺼져 있어도 EBS 볼륨, Elastic IP 등은 별도로 과금될 수 있다.

## 개인 프로젝트 관점에서의 주의점

트래픽이 거의 없는 개인 프로젝트에서 t3.micro 같은 저사양 인스턴스를 24시간 켜두면, 실제 사용량이 하루 몇 분에 불과해도 한 달 내내 요금이 발생한다. 즉 "항상 켜져 있는 비용"을 감수할 가치가 있는지가 EC2 선택의 핵심 판단 기준이다.
