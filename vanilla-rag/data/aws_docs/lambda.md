# AWS Lambda 개요

AWS Lambda는 서버를 프로비저닝하거나 관리할 필요 없이 코드를 실행하는 서버리스 컴퓨팅 서비스다. 요청이 들어올 때만 함수가 실행되고, 실행이 끝나면 리소스가 반환된다.

## 언제 적합한가

- 트래픽이 간헐적이거나 예측하기 어려운 워크로드(예: 개인 프로젝트, 이벤트 기반 처리, 웹훅 처리)에 적합하다.
- 요청이 없을 때는 과금되지 않으므로, 트래픽이 적은 서비스는 EC2보다 훨씬 저렴할 수 있다.
- 최대 실행 시간 15분 제한이 있어, 장시간 실행되는 배치 작업이나 상시 연결이 필요한 워크로드(WebSocket 서버 등)에는 부적합하다.

## 요금 구조

- 호출 횟수(요청 수)와 실행 시간(GB-초 단위, 할당 메모리 × 실행 시간)에 따라 과금.
- 매달 무료 사용량(Free Tier)이 있어 트래픽이 적은 개인 프로젝트는 사실상 무료로 운영 가능한 경우가 많다.
- 콜드 스타트(Cold Start): 오랜만에 호출되면 초기화 지연이 발생할 수 있다. 트래픽이 매우 낮은 서비스에서는 이 지연이 사용자 경험에 영향을 줄 수 있다.

## 개인 프로젝트 관점에서의 주의점

트래픽이 적은 개인 프로젝트, 특히 하루 몇 번 정도만 호출되는 API라면 Lambda가 비용 측면에서 가장 유리한 선택지인 경우가 많다. 다만 응답 지연에 민감한 서비스라면 콜드 스타트 영향을 검토해야 한다.
