# AWS Fargate 개요

AWS Fargate는 ECS 또는 EKS에서 컨테이너를 실행할 때 서버(EC2 인스턴스)를 직접 관리하지 않아도 되는 서버리스 컴퓨팅 엔진이다. 컨테이너 이미지를 배포하면 Fargate가 실행에 필요한 인프라를 자동으로 프로비저닝한다.

## 언제 적합한가

- 이미 Docker 컨테이너로 패키징된 애플리케이션을 서버 관리 부담 없이 운영하고 싶을 때 적합하다.
- Lambda의 15분 실행 시간 제한이나 특정 런타임 제약이 부담스러운 경우, 컨테이너 기반으로 더 긴 프로세스나 커스텀 런타임을 실행할 수 있다.
- 여러 컨테이너로 구성된 마이크로서비스 아키텍처를 운영할 때 EC2 클러스터 관리 없이 확장할 수 있다.

## 요금 구조

- 실행 중인 태스크(vCPU, 메모리 조합)에 대해 초 단위로 과금. Lambda처럼 완전한 요청 단위 과금은 아니며, 태스크가 떠 있는 동안 계속 과금된다.
- On-Demand와 Spot(Fargate Spot) 요금제가 있으며, Fargate Spot은 최대 70% 할인되지만 중단될 수 있다.
- EC2보다는 관리형이라 운영 부담이 적지만, Lambda보다는 상시 실행 비용이 발생하기 쉽다.

## 개인 프로젝트 관점에서의 주의점

트래픽이 매우 적은 개인 프로젝트에는 Fargate가 EC2보다는 낫지만 Lambda보다는 비용이 높을 수 있다. 다만 컨테이너 기반 배포가 이미 되어 있거나, Lambda의 실행 시간/런타임 제약을 피하고 싶다면 EC2보다 관리 부담이 적은 대안으로 고려할 만하다.
